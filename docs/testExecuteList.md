# テスト実行手順書 (統合検証用)

## 概要
本ドキュメントは、各環境における統合検証手順を定義します。
検証の効率化のため、**エンジニア**と**非エンジニア**の役割別にタスクを分類しています。

---

## 0. 役割別タスク分担

### 1.1 ステージング環境 (Vercel / Cloud Run)
- **Frontend (Vercel)**:web-pj-three.vercel.app(有効になり次第stg.wyze-project.com)
- **Backend (Cloud Run)**: https://backend-611370943102.us-east1.run.app
- **検証端末**: iPhone (Safari) / Android (Chrome) の実機ブラウザ。

### 🛠️ エンジニア (Developer / Infra)
- **すべきこと (Should)**:
  - インフラ（Cloud Run/Vercel）の設定・環境変数の整合性確認。
  - バックエンドログの監視と、エラー発生時の原因特定。
  - Webhook の到達確認および DB のレコード状態の直接確認。
- **できること (Can)**:
  - 全てのテスト項目。特にブラウザのデベロッパーツールを用いたデバッグ。

### 🎨 非エンジニア (PM / Business / Client)
- **すべきこと (Should)**: 
  - ユーザー視点での UI/UX（使いやすさ、文言の違和感）の最終チェック。
  - 決済フローが直感的に理解できるか、価格表示が正しいかの確認。
  - スマホ実機での操作感（指の届きやすさ等）の検証。
- **できること (Can)**: 
  - 管理画面（ログ等）以外の、ブラウザおよび実機で完結する全ての操作。

---

## 1. 環境接続・準備 【主にエンジニア】

### 1.1 インフラ整合性チェック (Should: エンジニア)
- [ ] **Backend (Cloud Run)**: `FRONTEND_URL` が `https://web-pj-three.vercel.app` になっているか。
- [ ] **Stripe**: ダッシュボードの Webhook 送信先が正しいか。
- [ ] **環境変数**: Vercel と Backend で 10 個の Price ID が一致しているか。
- [ ] **モード確認**: `MOCK_MODE=false` でデプロイされているか。

---

## 2. 新規登録 & 認証 【共同作業】

### 2.1 認証コードの取得と入力
1. **非エンジニア (Can)**: `/signup` でメール送信を行い、待機する。
2. **エンジニア (Should)**: バックエンドのログからコードを取得し、非エンジニアに伝える。
3. **非エンジニア (Should)**: 6桁のコードを入力し、登録が完了するか確認。
4. **【追加検証】有効期限チェック (Should: エンジニア)**:
    - [ ] 発行から 10 分経過したコードでエラー（Expired）になるか。
    - [ ] 無効なコードを連続入力した際に正しくブロックされるか。

### 2.2 ソーシャルログイン (Should: 非エンジニア)
- [ ] Google ログインボタンから正常にステージング環境へログイン・リダイレクトされるか。

---

## 3. アカウント管理 & プロフィール編集 【主に非エンジニア】

### 3.1 ユーザー情報更新 (Should: 非エンジニア)
- [ ] ニックネーム、メールアドレスの変更・保存。
- [ ] **重複チェック**: 既に登録済みのメールアドレスに変更しようとした際、適切にエラー（409 Conflict）が出るか。
- [ ] 更新後、リロードしても反映されているか。

---

## 4. 決済・プラン・履歴 【共同作業】

### 4.1 プラン表示と決済 (Should: 非エンジニア)
- [ ] 3 プラン（Light/Basic/Pro）の 3 段階価格が正しく表示されているか。
- [ ] **検証用価格 (100円)** で実際に決済を行い、完了画面が表示されるか。

### 4.2 バックグラウンド同期 (Should: エンジニア)
- [ ] 決済完了直後、DB の `plan_tier` が即座に更新されているか。
- [ ] **Webhook 監視**: `checkout.session.completed` だけでなく、`customer.subscription.deleted` (解約) 時にプランが `free` に戻るか。

### 4.3 履歴と領収書 (Should: 非エンジニア)
- [ ] **次回のご請求予定**: クレジットカード情報ブロックの直下に金額と日付が表示されているか。
- [ ] **未加入時の表示**: サブスクリプションがない場合、「次回のご請求はありません」等の適切な表示になっているか。
- [ ] **領収書**: 履歴のボタンを押し、Stripe の PDF が別タブで開くか。

---

## 5. UI/UX & 実機チェック 【主に非エンジニア】

### 5.1 外部コンテンツ表示 (Should: 非エンジニア)
- [ ] **Instagram 投稿**: `/post` 画面でプレースホルダーではなく実画像が表示されるか。
- [ ] **Google 口コミ**: `/review` 画面で実際のレビュー内容が表示されるか。

---

## 6. トラブルシューティング 【エンジニア】

### 6.1 異常時の確認手順 (Should: エンジニア)
- **決済後 localhost に飛ぶ**: `FRONTEND_URL` の設定を確認。
- **価格が 0 円になる**: 環境変数 `NEXT_PUBLIC_STRIPE_PRICE_ID_...` の読み込み失敗を確認。
- **SNSログインエラー**: Google Cloud Console のリダイレクト URI 設定を確認。
