# テスト実行手順書 (Staging環境検証用)

## 概要
本ドキュメントは、ステージング環境（Frontend: Vercel / Backend: Cloud Run）における全機能の統合検証手順を定義します。

---

## 1. ステージング接続情報

- **Frontend (Vercel)**: `https://web-pj-develop.vercel.app` (またはプロジェクトURL)
- **Backend (Cloud Run)**: `https://backend-611370943102.us-east1.run.app`
- **環境設定**: Vercel の環境変数 `NEXT_PUBLIC_API_URL` が上記のバックエンドURLを指していることを確認してください。

---

## 2. 統合検証フロー (一気通貫)

### 2.1 新規登録 & 動的メール認証
1. `/signup` から新規登録を開始。
2. ステージング環境の **Google Cloud Logs** または **GitHub Actions ログ** に出力される `[Verification Code]` を確認。
3. コードを入力し、ユーザー情報登録（ニックネーム等）を完了。
**期待結果**: 正常にログイン後のホーム画面が表示されること。

### 2.2 プロフィール編集
1. `/account` を開き、ニックネームとメールアドレスを変更して「保存」をクリック。
2. 画面リロード後も変更が維持されているか確認。
**期待結果**: DBが更新され、変更後の名前が表示されること。

---

## 3. Stripe 課金 & 領収書検証 (後日実施分)

### 3.1 決済とプラン更新
1. `/billing` からプランに申し込む（Stripe Checkout へ遷移）。
2. テストカードで決済を完了し、アプリに戻る。
**期待結果**: ユーザーの `plan_tier` が更新（例: Pro）され、機能制限が解除されること。

### 3.2 領収書 PDF 検証
1. 「お支払い履歴」の **「PDF」ボタン** をクリック。
2. **検証項目**:
    - [ ] 宛名がログインユーザーのメールアドレスになっているか。
    - [ ] 金額、日付、プラン名が履歴データと一致しているか。
    - [ ] 日本語が文字化け（豆腐文字）せず、正しく表示されているか。
**不備がある場合の対処**:
- 文字化け時: `frontend/utils/generateReceipt.ts` のフォントパスまたは Base64 埋め込みを修正。
- ズレ発生時: 同ファイル内の `doc.text()` の座標値を微調整し、再ビルド。

---

## 4. 外部 API 実データ検証 (OAuth)

### 4.1 Instagram 連携
1. `MOCK_MODE=false` であることを確認。
2. OAuth 認可フローを完了させ、`/post` 画面を表示。
**期待結果**: 自身（または検証用アカウント）の最新投稿が実データとしてリスト表示されること。

### 4.2 Google レビュー表示
1. Google Business Profile 連携を完了させ、`/review` 画面を表示。
**期待結果**: 実際の店舗に寄せられた最新の口コミが表示されること。

---

## 5. デプロイ不達・エラー時の確認
- **Frontend**: Vercel のデプロイログを確認し、Root Directory が `frontend` になっているか確認。
- **Backend**: Google Cloud Run のログを確認し、`PORT=8080` で Listen しているか、および `DATABASE_URL` が正しく Secret Manager から注入されているか確認。
