# テスト実行手順書 (Staging環境検証用)

## 概要
本ドキュメントは、ステージング環境（Frontend: Vercel / Backend: Cloud Run）における全機能の統合検証手順を定義します。
**※スマートフォン実機での検証を前提としたチェック項目を含みます。**

---

## 1. ステージング接続情報

- **Frontend (Vercel)**: `https://web-pj-develop.vercel.app`
- **Backend (Cloud Run)**: `https://backend-611370943102.us-east1.run.app`
- **検証端末**: iPhone (Safari) / Android (Chrome) の実機ブラウザ。

---

## 2. 統合検証フロー (スマホ実機確認)

### 2.1 新規登録 & 動的メール認証
1. スマホで `/signup` を開き、ステップ2まで進む。
2. **スマホ特有の確認**: 
    - [ ] 認証コード入力欄にフォーカスした際、数字キーボードが正しく表示されるか。
    - [ ] キーボードが表示された状態で「送信」や「次へ」ボタンが画面外に押し出され、スクロール不能にならないか。
3. ログで確認したコードを入力し、登録を完了。

### 2.2 プロフィール編集
1. `/account` を開き、情報を変更して「保存」。
2. **スマホ特有の確認**:
    - [ ] 入力フィールドが指でタップしやすい高さ（44px以上）を確保しているか。
    - [ ] 保存ボタンが親指で届く位置にあり、誤タップしにくい配置か。

---

## 3. Stripe 課金 & 領収書検証 (モバイル対応)

### 3.1 決済とプラン更新
1. `/billing` からプランに申し込む。
2. **スマホ特有の確認**:
    - [ ] Stripe Checkout 画面がスマホ最適化（Apple Pay / Google Pay の表示等）されているか。
    - [ ] 決済完了後、アプリへのリダイレクトがスムーズに行われるか。

### 3.2 領収書 PDF 検証 (実機ダウンロード)
1. 「お支払い履歴」の **「PDF」ボタン** をクリック。
2. **スマホ特有の確認**:
    - [ ] iOS (Safari) で「ファイル」アプリに保存、またはプレビューが表示されるか。
    - [ ] Android (Chrome) で通知領域にダウンロード完了が表示され、開けるか。
    - [ ] PDF 内の文字がスマホの画面でも判読可能か。

---

## 4. UI/UX モバイルチェックリスト

- [ ] **ナビゲーション**: サイドメニュー（またはハンバーガーメニュー）が片手で操作可能か。
- [ ] **スクロール**: 長いレポート画面や投稿一覧で、カクつき（ジャンク）がなくスムーズにスクロールできるか。
- [ ] **タップ精度**: 投稿の削除アイコンや口コミの返信ボタンなど、小さな要素が隣接する要素と干渉せずにタップできるか。
- [ ] **視認性**: 屋外や明るい場所でも、ボタンの文字色や背景色のコントラストが十分か。
