# リリース戦略
- セマンティックバージョニングを採用 (vX.Y.Z)
- mainブランチへのマージでリリース

## インフラ構成と最適化

### 1. Dockerイメージの最適化
- **マルチステージビルドの採用**:
  - `builder` ステージでビルドに必要なツール群を使用。
  - 最終ステージは `alpine:3.20` を採用し、イメージサイズを最小化。
  - バイナリ実行に必要な `ca-certificates` のみを含める。
- **軽量ベースイメージ**:
  - `golang:alpine` 系イメージを採用し、ビルド時間の短縮とセキュリティリスクの低減を図る。

### 2. 環境区分
- **Staging**: `develop` ブランチへの push をトリガーに Cloud Run へ自動デプロイ。
- **Production**: `main` ブランチへの PR マージ後に手動承認または自動デプロイ（今後の拡張）。

### 3. 可用性と監視
- **ヘルスチェック**:
  - 各サービスに `/health` エンドポイントを実装。
  - Docker Compose および Cloud Run でヘルスチェックを有効化し、異常検知時に自動再起動・トラフィック遮断を実施。
- **ログ監視**:
  - Google Cloud Logging を使用したエラーログの監視。
