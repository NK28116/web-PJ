# 手動テスト手順書

## 概要
本ドキュメントは、実装フェーズ（MVP/Mock）における「ホームタブ（ダッシュボード）」機能の手動テスト手順を定義します。
今回の実装はFigmaデザインの再現とモックデータのインタラクションが主目的であるため、目視による確認を推奨します。

---

## 前提条件
- 開発サーバーが起動していること (`npm run dev`)
- ブラウザで `http://localhost:3000` (または該当URL) にアクセス可能であること
- Google Chrome (DevTools使用推奨)

## テスト対象
- **HomeTemplate**: ホームタブ全体のレイアウト
- **DashboardSection**: ダッシュボード統計情報の表示・切り替え
- **AccountLinkingSection**: アカウント連携トグル機能
- **ShopListSection**: 店舗リスト表示

---

## テスト項目一覧

### 1. 初期表示確認（未連携状態）
| No | 項目名 | 操作 | 期待値 (Expected Result) | 判定 |
|----|--------|------|--------------------------|------|
| 1-1 | ダッシュボード全体 | 画面ロード | ・ヘッダー「ダッシュボード」「更新する」が表示される<br>・「AIによる最優先アクション」セクションが表示され、内容が空（"-"）である | |
| 1-2 | オーバーレイ表示 | 画面ロード | ・統計情報（システム稼働状況など）の上に「Googleで連携すると使用できます」というオーバーレイが表示されている<br>・背面の統計情報がブラー（ぼかし）処理されている | |
| 1-3 | アカウント連携セクション | 画面ロード | ・「アカウント連携」カードが表示される<br>・Googleアイコン、「Google」、「未連携」、「連携する」ボタンが表示される<br>・Instagramアイコン、「Instagram」、「未連携」、「連携する」ボタンが表示される<br>・下部に「アカウントを追加」ボタンが表示される | |
| 1-4 | 店舗一覧 | 画面ロード | ・「店舗一覧」セクションが表示される<br>・モックデータ（ID: 0001, wyzesystem...）のカードが表示される<br>・ステータスが「連携待ち」（青丸）である | |

### 2. インタラクション確認（アカウント連携）
| No | 項目名 | 操作 | 期待値 (Expected Result) | 判定 |
|----|--------|------|--------------------------|------|
| 2-1 | Google連携ON | 「Google」行の「連携する」ボタンをクリック | ・ボタン表示が「解除する」（グレー背景）に変わる<br>・ステータス表示が「連携済み」に変わる<br>・**ダッシュボードのオーバーレイが消える**<br>・統計情報（システム稼働状況: 正常稼働中、数値など）が鮮明に表示される | |
| 2-2 | Google連携OFF | 「Google」行の「解除する」ボタンをクリック | ・ボタン表示が「連携する」（緑背景）に戻る<br>・ステータス表示が「未連携」に戻る<br>・**ダッシュボードに再度オーバーレイが表示される**<br>・統計情報が隠れる | |
| 2-3 | Instagram連携ON | 「Instagram」行の「連携する」ボタンをクリック | ・ボタン表示が「解除する」に変わる<br>・ステータス表示が「連携済み」に変わる<br>・(注: 今回の実装ではダッシュボード統計はGoogle連携のみに依存するため、統計情報は変化しないことを確認) | |
| 2-4 | Instagram連携OFF | 「Instagram」行の「解除する」ボタンをクリック | ・ボタン表示が「連携する」に戻る | |

### 3. デザイン・レスポンシブ確認
| No | 項目名 | 操作 | 期待値 (Expected Result) | 判定 |
|----|--------|------|--------------------------|------|
| 3-1 | コンポーネント構造 | 目視確認 | ・GoogleとInstagramが**単一のカード（枠線内）**に収まっていること（別々のカードに分かれていない）<br>・各セクション間の余白が均一であること | |
| 3-2 | フォント・スタイル | 目視確認 | ・フォントサイズ、色、ボタンのスタイルがFigmaデザインに近いこと（"Close enough"ではなく厳密に） | |
| 3-3 | スマホ表示 | ブラウザ幅を縮小 (モバイルサイズ) | ・レイアウトが崩れず、横スクロールが発生しないこと<br>・文字が見切れていないこと | |
