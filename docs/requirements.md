# 要件

## 対象

お支払い情報画面を Figma（docs/figma/Billing.png）と視覚的一致にする

対象：
components/templates/BillingTemplate/BillingTemplate.tsx

---

## 機能要件

### 1. ヘッダー

- 左上に「＜ お支払い情報」を表示
- タップで直前画面へ戻る

---

### 2. 登録クレジットカード情報表示

表示項目：
- カードブランド
- 下4桁（例：**** **** **** 1234）
- 有効期限

操作：
- 「カード情報を変更する」ボタンを表示
- タップでカード編集モーダルを表示（UIのみ実装・API未接続）

制約：
- フロントでカード番号フル保持しない
- カード番号は常にマスク表示
- モックデータを使用
- 表示ロジックとデータ取得ロジックを分離

---

### 3. お支払い履歴

表示項目（一覧形式）：
- 支払い年月日（YYYY/MM/DD）
- 金額（¥ + 3桁カンマ）
  
操作：
- 履歴一覧の上部または下部に「領収書 / PDF」ボタンを1つのみ配置（一覧一括出力）
- 押下で履歴全件を対象にPDF出力処理を呼び出す（モック実装可）

---

### 4. 次回のお支払い表示

- お支払い履歴セクションの下部中央に配置
- グレーアウト表示とする
- 表示形式：
  「次回のお支払い：YYYY/MM/DD ¥XXX,XXX」
- データ未存在時は
  「次回のお支払い予定はありません」と表示

---

## 信頼性要件

- null / undefined / NaN を画面表示しない
- 金額は必ず数値フォーマット（¥ + 3桁カンマ）
- 日付は YYYY/MM/DD 固定フォーマット
- モックデータ未取得時は空配列で描画崩れを起こさない

---

## 制限

- スマートフォン表示のみ対応
- 横スクロール禁止
- API未接続（モック使用）
