# 実装レビュー (Phase 7 完了 & Phase 8 実機連携設計)

## レビュー日: 2026-03-15
## レビュアー: 河崎 紗夜

### 1. 総評
Phase 7「Stagingリリース」の主要課題であった Vercel ビルドエラーが解消され、`https://web-pj-three.vercel.app/` にてフロントエンドが正常に配信されていることを確認しました。また、これに続く Phase 8「実機連携 & 最終検証」に向けた設計および実装指示書（`docs/task-to-claude.md`）の策定が完了しており、MVP（Minimum Viable Product）としての完成に向けた盤石な体制が整っています。

---

### 2. 項目別詳細レビュー

#### A. ステージング環境の正常化 (✅ 完了)
- **Vercel 設定**: モノレポ構成特有の Root Directory 問題が解消され、フロントエンドが正しくビルドされています。
- **環境変数**: `NEXT_PUBLIC_API_URL` が Cloud Run のバックエンド URL に正しく紐付けられ、ブラウザからの API 通信が可能な状態です。

#### B. Phase 8 実機連携設計 (✅ 優秀)
- **API 統合**: Google ビジネスプロフィールの口コミ取得・返信、および Instagram の投稿表示が具体的に計画されており、モックからの脱却が明確に定義されています。
- **決済・領収書フロー**: Stripe Checkout への遷移だけでなく、日本国内の商習慣（MoneyForward 形式）に合わせた領収書 PDF 生成までをスコープに含めている点は、実運用を強く意識した優れた設計です。

#### C. セキュリティとインフラ設定 (⚠️ 注意事項)
- **OAuth リダイレクト**: Vercel のドメイン確定に伴い、GCP および Meta コンソールでの「承認済みのリダイレクト URI」の更新が必須となります。これを行わないと、ステージング環境でのログイン連携が失敗するため、実装時の最優先事項としてマークしています。

### 3. ユーザーによる確認方法 (Validation Guide)

実装が進むにつれ、以下の手順で動作を確認してください。

#### ① ステージング環境の表示確認
- URL: `https://web-pj-three.vercel.app/`
- **確認内容**: 画面が正しく表示され、デベロッパーツールの「Network」タブで、バックエンド API (`backend-...run.app`) への通信（ログイン試行等）が 401 や 200 で応答しているか。

#### ② Google/Instagram 連携の検証 (Phase 8 実装後)
- **手順**: 「設定」または「連携」画面から各プロバイダーのボタンを押し、認可フローを完了させる。
- **確認内容**: ダッシュボードやレポート画面に、モックデータではなく **実際のアカウントの投稿画像や口コミ内容** が表示されるか。

#### ③ Stripe 決済と領収書の検証 (Phase 8 実装後)
- **手順**: 料金ページからプランを選択し、Stripe Checkout 画面へ遷移。テスト用カード (`4242...`) で決済を実行。
- **確認内容**:
  - 決済完了後、アプリ内のステータスが「有料プラン」に変わるか。
  - 「領収書 / PDF」ボタンを押し、`docs/Receipt/` のテンプレートに近い形式で PDF がダウンロードされるか。
