# Claudeへの実装指示書: 複数アカウント環境の構築と基盤要件の対応

`docs/requirements.md` が更新されました。本要件の最優先事項は **「複数アカウント環境でデータ分離が完全に保証されること」** です。UI改善やパフォーマンス最適化はスコープ外とします。以下の要件に従い、実装を進めてください。

## 1. 複数アカウント動作とデータ分離機能の実装 (Priority: Highest - Issue #9)
- **データ分離の仕組み**: APIレイヤーで `user_id` によるデータのフィルタリングを強制し、他のユーザーのデータ（投稿や情報のリストなど）が絶対に混在・アクセスされないように実装してください。
- **JWT検証の厳格化**: JWTの署名検証を必須とし、アルゴリズムの固定化（`alg=none`の禁止）、トークン失効時や改ざん時の適切なエラーレスポンス（401等）を実装してください。
- **権限管理**: 管理者アカウントと一般ユーザーアカウントでのアクセス権限制御を明確にし、管理者権限の誤使用が発生しないようにシステムを保護してください。

## 2. ローカル開発環境の整備とシードデータ (Priority: High)
- **Docker Composeの改修**:
  - 本番と同バージョンの PostgreSQL 15 を指定し、永続ボリュームとDBヘルスチェックを追加してください。
  - バックエンド(Go/Gin)のコンテナは、データベースがhealthy状態になるまで起動を待機する制御（`depends_on` の `condition: service_healthy` など）を追加してください。
- **シードデータの自動生成**:
  - `管理者アカウント`, `一般ユーザーA`, `一般ユーザーB` を手動登録なしで自動的に生成するシード（seed）処理を実装してください。
- **開発用ログインショートカット実装**:
  - シードで作成したアカウントでのログインを容易にするため、`components/templates/LoginTemplate/index.tsx` の既存の「[DEV]開発用ログイン」ボタンと同様の形式で、以下の3つの開発環境限定ボタンを作成してください。
    - `[DEV]開発用ログイン(管理者)`
    - `[DEV]開発用ログイン(一般ユーザーA)`
    - `[DEV]開発用ログイン(一般ユーザーB)`

## 3. データベースマイグレーションの導入 (Priority: High)
- `golang-migrate` あるいは `Prisma migrate` を用いたマイグレーション管理を実装し、スキーマ差分事故を防止する構成に変更してください。
- すでに存在する場合を除き、`AutoMigrate` のみに依存して本番運用を行うような実装は排除してください。

## 4. CI/CDパイプラインへのテスト環境適応 (Priority: Medium)
- GitHub Actions のPR時のテスト（CI）実行環境において、テスト用のPostgreSQLを起動するように設定を追加してください。
- CI内でマイグレーションダウン/アップを実行し、データベースの準備が整った状態でテスト(Go側)が走るように Action を修正してください。

---
**セキュリティおよび設定等の注意事項**:
- パスワードのハードコード、および `JWT_SECRET` 等のGit管理は厳禁です。必ず環境変数（`.env` 経由）から注入してください（最低32文字以上）。
- 分離保証のテストとして「同時ログイン（別ブラウザからの擬似など）」「ログアウト後の別ユーザー再ログイン」「改ざんトークンの拒否」等の複数アカウント前提の動作シナリオが通るよう検証してください。
